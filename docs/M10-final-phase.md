## Blocker 1: Debug release.yml Startup Failure

### Investigation

1. Visit https://github.com/JAS0N-SMITH/redactlog/actions
2. Click the most recent "Release" workflow run
3. Expand the failed step and read error message
4. Common causes:
   - `goreleaser/goreleaser-action@v6` not found → check action syntax
   - `GITHUB_TOKEN` permission insufficient → check `permissions: contents: write`
   - Go version unavailable → `actions/setup-go@v5` may have failed upstream
   - Checkout issue → `fetch-depth: 0` was rejected

### Fix Workflow

If you identify the issue:

1. Edit `.github/workflows/release.yml` with the fix
2. Commit: `git add .github/workflows/release.yml && git commit -m "ci: fix release.yml [description of issue]"`
3. Create a test tag to validate:
   ```bash
   git tag v1.0.0-rc.1-test
   git push origin v1.0.0-rc.1-test
   ```
4. Monitor the Release workflow. Once it succeeds:
   ```bash
   git push origin --delete v1.0.0-rc.1-test
   git tag -d v1.0.0-rc.1-test
   ```

### Alternative (Manual Release)

If the workflow remains broken and you need to publish immediately:

1. Install goreleaser locally: https://goreleaser.com/install/
2. Run: `goreleaser release --clean` from the repo root
3. Manual GitHub Release creation via `gh release create` with artifacts
